When supporting multiple currencies in a payment system that uses Stripe, keep the internal ledger in a single canonical currency (USD cents in this case). Currency conversion should happen only at the Stripe payment boundary — when creating PaymentIntents. Store the original canonical-currency amount in PaymentIntent metadata (e.g. `originalUsdCents`) so that webhook handlers can always credit the correct amount regardless of which currency the PaymentIntent was denominated in. This avoids reverse-conversion rounding errors and keeps all internal accounting (balances, bets, payouts) in one currency. Legacy PaymentIntents without the metadata field should fall back to assuming the canonical currency.
