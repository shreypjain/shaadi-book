Cursor-based pagination in tRPC queries follows a consistent pattern: take `limit + 1` rows, pop the extra to derive `nextCursor`, and return `{ items, nextCursor }`. The cursor is the ID of the last item. On the Prisma side, pass `cursor: { id: input.cursor }, skip: 1` to start after the cursor. The frontend should lazy-load on first expand and offer a 'Load more' button that passes the cursor back. This avoids offset-based pagination issues with changing data (new trades appearing).
