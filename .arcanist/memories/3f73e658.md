When an admin action (like approving a suggestion) triggers downstream side effects (like creating a market and seeding liquidity), wrap those side effects in try/catch so the primary operation (the approval) always succeeds. The pattern is: perform the core DB write first, then attempt secondary operations with errors caught and logged. For multi-step side effects (create market → fetch outcomes → seed liquidity), each step can fail independently. Use fire-and-forget (.catch()) for truly non-critical trailing steps like house seeding, while logging errors for observability. This prevents a seeding failure from rolling back an otherwise successful approval.
