When committing directly to main by mistake instead of a feature branch, the fix is: create the feature branch at the current commit (`git push origin HEAD:feat/branch-name`), then reset main back (`git reset --hard HEAD~1 && git push origin main --force`), then create the PR from the feature branch. This ensures the PR shows a proper diff against main. Avoid force-pushing main if others may have pulled — coordinate first.
