When adding a new Prisma model, the Prisma client needs to be regenerated (`npx prisma generate`) before TypeScript can recognize the new model's types. Without regeneration, any code referencing the new model (e.g., `prisma.marketWatch.findMany(...)`) will produce implicit `any` errors. If you can't regenerate in your environment, add explicit type annotations to work around the missing types. This is especially easy to miss when the new model is only used in a few places and the rest of the codebase compiles fine.
