When disabling PostgreSQL triggers inside a Prisma `$transaction` (e.g., `ALTER TABLE ... DISABLE TRIGGER ALL`), you must re-enable them before the transaction commits, even in error paths. Use a try/finally pattern around the core logic. This is critical for immutability triggers on financial tables (purchases, transactions) — if triggers aren't re-enabled due to an error, subsequent operations outside the transaction will bypass integrity checks. The serializable isolation level should also be used for operations that modify balances and positions to prevent race conditions.
