When adding new enum values to a Prisma enum that maps to a PostgreSQL enum type, you must create a migration with `ALTER TYPE "EnumName" ADD VALUE IF NOT EXISTS 'NEW_VALUE'`. Prisma's default migration generation may try to recreate the enum which can fail if rows reference it. The `IF NOT EXISTS` guard makes the migration idempotent and safe to re-run.
