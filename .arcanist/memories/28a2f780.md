Non-critical side effects in mutation handlers (auto-watch on purchase, SMS notifications, WebSocket broadcasts) should be fire-and-forget with `.catch()` to log warnings rather than failing the main operation. This pattern ensures the primary transaction (e.g., a purchase) succeeds even if secondary effects like notifications or watchlist updates fail. Use `console.warn` with a descriptive prefix like `[market.buy] Auto-watch upsert failed (non-fatal):` so failures are visible but non-blocking.
