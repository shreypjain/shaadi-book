In this codebase, the `Purchase` table records buy-side entries only (sales are tracked in the `transactions` table and do NOT create Purchase rows). When aggregating user positions from purchases, summing all Purchase rows gives the correct gross buy position. Do not use the `Position` table for this purpose — it reflects net-of-sells mutable state, which is a different semantic. The distinction matters for leaderboard/positions endpoints where gross deployment is the intended metric.
