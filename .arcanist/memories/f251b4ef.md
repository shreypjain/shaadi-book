When defining callbacks in React components that are referenced by `useEffect` hooks, the callback must be declared (or defined via `useCallback`) before the `useEffect` that references it. JavaScript's temporal dead zone (TDZ) means that even though `const`/`let` declarations are hoisted, they cannot be accessed before their declaration line executes. This is especially easy to miss in large component files where multiple `useEffect` blocks and callbacks are interspersed. The fix is to reorder declarations so that any callback referenced in a `useEffect` dependency array or body appears above that effect in the source.
